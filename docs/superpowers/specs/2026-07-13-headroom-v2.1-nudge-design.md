# headroom-usage-indicator v2.1.0 — Missed-Opportunity Nudge

Date: 2026-07-13
Status: Approved

## Goal

The red badge says "not compressing" but not whether there was anything worth
compressing. v2.1 counts large tool results that were never compressed and
surfaces the count on the idle badges, turning the indicator from a passive
gauge into an actionable nudge.

## Detection

In `scripts/statusline.sh`'s `compute()` (so the result rides the existing
per-session size-keyed cache):

- A new jq pass counts `tool_result` blocks whose text content totals
  ≥ `NUDGE_BYTES` (hardcoded script variable, `4096`).
- `content` may be a plain string or an array of blocks — measure the string's
  length, or the summed length of the array's `.text` fields.
- Results whose `tool_use_id` links to a `tool_use` named `mcp__headroom__*`
  (any headroom tool) are **excluded** — a large `headroom_retrieve` result is
  headroom working as intended, not a miss.
- Forgiveness heuristic: `missed = max(0, big_count − compress_count)` —
  compressing a blob doesn't remove its original from the transcript, so each
  compression forgives one big blob. Computed in shell from the jq counts.

## Cache format

`size|n|saved|last_ts|missed` (5th field appended). A hit now also requires
the 5th field to be non-empty, so a v2.0 4-field cache line recomputes once
and is rewritten in the new format — no migration.

## Display

`missed` appears only when > 0; the green active badge never shows it.

| State | Badge |
|---|---|
| red, missed > 0 | `○ headroom idle · 4 big blobs uncompressed` (singular-aware: `1 big blob uncompressed`) |
| red, missed = 0 | unchanged: `○ headroom idle (not compressing yet)` |
| grey, missed > 0 | `○ headroom idle · ~2.4k tok · $0.007 · 3× · 2 missed | $1.83 all-time` (` · N missed` before the lifetime segment) |
| grey, missed = 0 | unchanged |
| green | unchanged |

## Tests (test.sh, +5 groups)

1. One >4 KB tool_result, no compress → red badge `1 big blob uncompressed`;
   two such blobs → `2 big blobs uncompressed` (plural).
2. Blob just under 4096 bytes → red stays `(not compressing yet)`.
3. Oversized result linked to a `mcp__headroom__headroom_retrieve` tool_use →
   excluded; red stays `(not compressing yet)`.
4. Forgiveness: 2 big blobs + 1 old-timestamp compress → grey badge with
   `· 1 missed`; 1 big blob + 1 compress → no ` missed` segment.
5. Cache upgrade: a hand-written 4-field cache line for the session → render
   recomputes (badge correct) and the cache file is rewritten with 5 fields.

## Error handling

Same rules as v2.0: all new jq/arithmetic guarded; any failure degrades to
`missed=0` (no nudge) — the badge never renders broken.

## Out of scope

- Structured-ness detection (JSON vs code) — size-only heuristic for now.
- Env-var threshold override (decided: hardcoded).
- Nudging on the green active badge (decided: never).
