# headroom-usage-indicator v2.2.0 — "Dangi" Real-Time Detector

Date: 2026-07-13
Status: Approved

## Goal

The v2.1 `missed` counter is retrospective. Dangi is a real-time detector: a
Claude Code **PostToolUse hook** that fires the moment a large uncompressed
tool result lands, plus a mascot on the status line. Three channels (all
approved by the user): nudge Claude in-context, macOS notification to the
user, and a status-line mascot.

## Component 1 — `scripts/dangi-hook.sh` (PostToolUse hook)

Contract verified against https://code.claude.com/docs/en/hooks.md
(2026-07-13): stdin JSON carries `tool_name`, `tool_output` (string or
object — the hook's `tostring` handles both; very large outputs arrive
truncated, per docs ~10,000 chars with file-reference replacement),
`session_id`; stdout JSON
`{"hookSpecificOutput":{"hookEventName":"PostToolUse","additionalContext":"…"}}`
injects a system-reminder next to the tool result; matcher `*` matches all
tools incl. MCP; hooks merge additively across settings sources; parallel,
non-blocking.

Behavior (always `exit 0`; every failure path silent):

1. Skip when `tool_name` starts with `mcp__headroom__` (headroom's own
   activity) or `tool_output` length < `NUDGE_BYTES=4096` (same threshold as
   the badge).
2. **Claude nudge** — emit `additionalContext`:
   `🤖 Dangi: that <tool> output was ~<N> KB and was not compressed — if it
   is structured/repetitive, consider mcp__headroom__headroom_compress to
   save context.` Rate-limited: at most once per `NUDGE_COOLDOWN=60` s per
   session.
3. **macOS notification** — `osascript display notification` titled
   `🤖 Dangi`, at most once per `NOTIFY_COOLDOWN=300` s per session; fired in
   the background; skipped when `osascript` is absent (Linux) or
   `DANGI_NO_NOTIFY` is set (tests).
4. Rate-limit state: `$STATE_DIR/session-<sid>.dangi` containing
   `last_nudge_epoch last_notify_epoch` (numeric-validated on read).

## Component 2 — status-line mascot (`scripts/statusline.sh`)

Appended to the right end of every badge state:

- `missed = 0`: `  😴 dangi` (dim, `\033[2m`)
- `missed > 0`: `  🤖 dangi: N!` (yellow, `\033[33m`)

The mascot appears on all three badge states (including green — it is a
separate companion element; the v2.1 rule that green never shows the inline
` · N missed` segment is unchanged).

## Component 3 — installer (SKILL.md)

The installer additionally:
- copies `scripts/dangi-hook.sh` → `~/.claude/dangi-hook.sh` (755, re-copied
  on rerun);
- merge-appends into settings `hooks.PostToolUse` the entry
  `{"matcher": "*", "hooks": [{"type": "command", "command": "bash \"<home>/.claude/dangi-hook.sh\"", "timeout": 10}]}`
  unless any existing PostToolUse entry already references `dangi-hook.sh`
  (idempotent; existing user hooks preserved).

## Tests

Hook (driven by piping synthetic hook-input JSON, `DANGI_NO_NOTIFY=1`,
temp `HEADROOM_STATE_DIR`):
1. Big (4096-char) Bash output → stdout contains `additionalContext` and
   `Dangi`; exit 0.
2. Immediate second call, same session → empty stdout (cooldown); exit 0.
3. Different session immediately → nudges (cooldown is per-session).
4. 4095-char output → empty stdout.
5. Big `mcp__headroom__headroom_compress` output → empty stdout.
6. Garbage stdin → empty stdout, exit 0.

Mascot: badge contains `🤖 dangi: 1!` when missed = 1; contains `😴 dangi`
when missed = 0.

## Known limitations (documented, accepted)

- Very large outputs reach hooks truncated (per docs ~10,000 chars with
  file-reference replacement), so a huge blob may evade the real-time ping;
  the transcript-based `missed` counter still catches it.
- Hook cannot know whether the blob was *later* compressed — the badge's
  forgiveness math remains the source of truth; Dangi only detects arrival.

## Out of scope

- Linux desktop notifications (notify-send) — macOS only for now.
- Configurable cooldowns via env — hardcoded like NUDGE_BYTES.
