# headroom-usage-indicator v2.2.0 ("Dangi") Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A real-time detector ("Dangi"): a PostToolUse hook that nudges Claude in-context and pings the user via macOS notification the moment a large uncompressed tool result lands, plus a 😴/🤖 mascot on the status line.

**Architecture:** New standalone `scripts/dangi-hook.sh` reads the PostToolUse stdin JSON and emits `additionalContext` JSON (rate-limited per session via a small state file); `scripts/statusline.sh` appends a mascot segment driven by the existing `missed` value; the SKILL.md installer copies the hook and merge-appends a `hooks.PostToolUse` entry.

**Tech Stack:** bash, jq, awk, osascript (optional). Repo `/Users/abhi/Desktop/headroom-plugin`; `./test.sh` currently 33 checks.

## Global Constraints

- The hook must **always exit 0** and never emit anything on failure paths — a broken hook disturbs every tool call of every session. All jq/read/arithmetic guarded; numeric fields validated with a `case … (*[!0-9]*|"")` pattern before arithmetic.
- Thresholds/cooldowns hardcoded: `NUDGE_BYTES=4096`, `NUDGE_COOLDOWN=60` (context nudges), `NOTIFY_COOLDOWN=300` (notifications).
- Skip all tools whose name starts with `mcp__headroom__`.
- Notifications: only when `osascript` exists AND `DANGI_NO_NOTIFY` is unset; fired with `&` (never block); stdout/stderr discarded.
- Hook output when nudging must be EXACTLY one JSON object on stdout: `{"hookSpecificOutput":{"hookEventName":"PostToolUse","additionalContext":"…"}}` — nothing else may reach stdout (a stray echo corrupts the JSON).
- Rate-limit state file: `$STATE_DIR/session-<sid>.dangi`, one line `last_nudge_epoch last_notify_epoch`; `STATE_DIR="${HEADROOM_STATE_DIR:-$HOME/.claude/headroom-indicator}"`.
- Mascot: `missed = 0` → `  😴 dangi` in dim (`\033[2m`); `missed > 0` → `  🤖 dangi: N!` in yellow (`\033[33m`); appended to ALL badge states. The v2.1 inline ` · N missed` grey segment is unchanged.
- The badge must never render broken; macOS + Linux portability preserved.
- Version 2.2.0 in both `.claude-plugin/plugin.json` and `.claude-plugin/marketplace.json`.
- Branch `feature/v2.2-dangi` (created by the controller). Commit after every task.

- [ ] **Step 3: Write the hook**

Create `scripts/dangi-hook.sh` (mode 755):

```bash
#!/usr/bin/env bash
# dangi — real-time compression detector for the headroom-usage-indicator plugin.
# Registered as a Claude Code PostToolUse hook (matcher "*"). Reads the hook
# input JSON on stdin; when a tool result >= NUDGE_BYTES lands and wasn't
# produced by headroom, nudges Claude via additionalContext (rate-limited)
# and pings the user with a macOS notification (rate-limited harder).
# MUST always exit 0 and print nothing except the single JSON nudge —
# anything else disturbs every tool call of every session.

set -u

NUDGE_BYTES=4096       # tool outputs at least this large are compression candidates
NUDGE_COOLDOWN=60      # seconds between context nudges per session
NOTIFY_COOLDOWN=300    # seconds between macOS notifications per session
HPREFIX="mcp__headroom__"
STATE_DIR="${HEADROOM_STATE_DIR:-$HOME/.claude/headroom-indicator}"

in=$(cat)

tool=$(printf '%s' "$in" | jq -r '.tool_name // empty' 2>/dev/null) || exit 0
[ -n "$tool" ] || exit 0
case "$tool" in "$HPREFIX"*) exit 0 ;; esac

size=$(printf '%s' "$in" | jq -r '.tool_output // "" | tostring | length' 2>/dev/null) || exit 0
case "$size" in (*[!0-9]*|"") exit 0 ;; esac
[ "$size" -ge "$NUDGE_BYTES" ] || exit 0

sid=$(printf '%s' "$in" | jq -r '.session_id // "unknown"' 2>/dev/null) || sid="unknown"
[ -n "$sid" ] || sid="unknown"
now=$(date +%s)
kb=$(( size / 1024 ))

state="$STATE_DIR/session-$sid.dangi"
last_nudge=0; last_notify=0
if [ -f "$state" ]; then
  read -r last_nudge last_notify < "$state" 2>/dev/null || true
fi
case "$last_nudge" in (*[!0-9]*|"") last_nudge=0 ;; esac
case "$last_notify" in (*[!0-9]*|"") last_notify=0 ;; esac

if [ -z "${DANGI_NO_NOTIFY:-}" ] && command -v osascript >/dev/null 2>&1 \
   && [ $(( now - last_notify )) -ge "$NOTIFY_COOLDOWN" ]; then
  osascript -e "display notification \"A ${kb} KB ${tool} output just landed uncompressed — headroom could shrink it.\" with title \"🤖 Dangi\"" >/dev/null 2>&1 &
  last_notify=$now
fi

nudge=0
if [ $(( now - last_nudge )) -ge "$NUDGE_COOLDOWN" ]; then
  nudge=1
  last_nudge=$now
fi

if mkdir -p "$STATE_DIR" 2>/dev/null; then
  printf '%s %s\n' "$last_nudge" "$last_notify" > "$state" 2>/dev/null || true
fi

if [ "$nudge" -eq 1 ]; then
  printf '{"hookSpecificOutput":{"hookEventName":"PostToolUse","additionalContext":"🤖 Dangi: that %s output was ~%s KB and was not compressed — if it is structured/repetitive, consider mcp__headroom__headroom_compress to save context."}}' "$tool" "$kb"
fi
exit 0
```

Then: `chmod +x scripts/dangi-hook.sh`

(Tool names are `[A-Za-z0-9_]` so interpolating `%s` into the JSON is safe; nothing else user-controlled reaches the output.)
